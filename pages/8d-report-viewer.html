문제 상황: 사용자 인증 시스템에서 로그인 시도 시 세션 토큰 생성 과정에서 간헐적으로 NullPointerException이 발생하여 사용자가 로그인할 수 없는 상황이 발생합니다.


재현 단계:

  1. 사용자가 로그인 페이지에서 올바른 credentials을 입력
  2. 서버에서 사용자 인증 프로세스 시작
  3. JWT 토큰 생성 과정에서 user context가 null인 경우 발생
  4. 500 Internal Server Error 반환

예상 결과 vs 실제 결과:

• 예상: 정상적인 JWT 토큰 생성 및 로그인 완료
• 실제: NullPointerException 발생으로 로그인 실패

적용된 응급 조치:

  • JWT 토큰 생성 로직에 null 체크 로직 추가
  • User context 초기화 실패 시 기본값 설정
  • 에러 핸들링 강화로 graceful degradation 구현
  • 로그인 실패 시 사용자에게 명확한 에러 메시지 제공

🎯
D4: 근본 원인 분석
5-Why 분석을 통한 원인 파악

분석 방법: 5-Why 분석 및 코드 리뷰


근본 원인:

사용자 세션 초기화 과정에서 데이터베이스 커넥션 풀 부족으로 인한 user context 로딩 실패가 주 원인입니다. 동시 사용자 수가 증가할 때마다 커넥션 풀이 고갈되어 새로운 사용자의 context 정보를 가져올 수 없는 상황이 발생합니다.


5-Why 분석 과정:

  1. Why 1: 로그인이 실패하는가? → JWT 토큰 생성 시 NullPointerException 발생
  2. Why 2: JWT 토큰 생성에서 NPE가 발생하는가? → User context가 null
  3. Why 3: User context가 null인가? → DB에서 사용자 정보 로딩 실패
  4. Why 4: DB에서 정보 로딩이 실패하는가? → DB 커넥션을 얻을 수 없음
  5. Why 5: DB 커넥션을 얻을 수 없는가? → 커넥션 풀 크기가 부족하고 적절한 타임아웃 설정 없음

기여 요인:

  • 커넥션 풀 최대 크기가 10으로 설정되어 있어 동시 사용자 증가에 대응 부족
  • 커넥션 타임아웃 설정이 없어 무한 대기 발생
  • 에러 핸들링 로직 부족으로 예외 상황에서의 적절한 처리 미흡
