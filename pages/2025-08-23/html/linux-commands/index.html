🚀 기초 명령어

Linux를 시작하는 모든 사용자가 반드시 알아야 할 핵심 명령어들입니다.

pwd

기초
현재 작업 디렉토리의 전체 경로를 출력합니다. 터미널에서 현재 위치를 파악할 때 사용하는 가장 기본적인 명령어입니다.
예시:
$ pwd
/home/username/Documents
팁: Print Working Directory의 줄임말입니다.

ls

기초
디렉토리 내용을 목록으로 보여줍니다. Linux에서 가장 자주 사용되는 명령어 중 하나로, 파일과 디렉토리의 정보를 다양한 형태로 확인할 수 있습니다.
기본 사용법:
$ ls
Desktop  Documents  Downloads  Pictures
옵션 활용:
$ ls -la
total 32
drwxr-xr-x  5 user user 4096 Aug 23 10:30 .
drwxr-xr-x  3 user user 4096 Aug 22 15:20 ..
-rw-r--r--  1 user user  220 Aug 22 15:20 .bashrc
-l: 자세한 정보 표시
-a: 숨겨진 파일 포함
-h: 사람이 읽기 쉬운 크기 표시
-t: 수정 시간순 정렬

cd

기초
디렉토리를 변경합니다. 파일 시스템을 탐색하고 원하는 위치로 이동할 때 사용합니다. Change Directory의 줄임말입니다.
다양한 사용법:
# 특정 디렉토리로 이동
$ cd /home/user/Documents

# 홈 디렉토리로 이동
$ cd ~
$ cd

# 상위 디렉토리로 이동
$ cd ..

# 이전 디렉토리로 이동
$ cd -
유용한 단축키:
  • ~: 홈 디렉토리
  • ..: 상위 디렉토리
  • -: 이전 디렉토리

📁 파일 시스템 관리

mkdir

기초
새로운 디렉토리를 생성합니다. Make Directory의 줄임말로, 프로젝트 구조를 만들거나 파일을 체계적으로 정리할 때 사용합니다. 
# 단일 디렉토리 생성
$ mkdir new_folder

# 여러 디렉토리 동시 생성
$ mkdir folder1 folder2 folder3

# 중간 경로까지 생성 (-p 옵션)
$ mkdir -p path/to/deep/folder

# 권한을 지정하여 생성
$ mkdir -m 755 secure_folder

# 상세한 출력과 함께 생성
$ mkdir -v new_project

touch

기초
빈 파일을 생성하거나 파일의 타임스탬프를 수정합니다. 개발 과정에서 새 파일을 빠르게 만들거나, 파일의 수정 시간을 현재 시간으로 갱신할 때 사용합니다. 
# 새 파일 생성
$ touch newfile.txt

# 여러 파일 동시 생성
$ touch file1.txt file2.txt file3.txt

# 특정 시간으로 타임스탬프 설정
$ touch -t 202312251200 oldfile.txt

# 다른 파일과 같은 타임스탬프로 설정
$ touch -r reference.txt newfile.txt

cp

기초
파일이나 디렉토리를 복사합니다. 
# 파일 복사
$ cp source.txt destination.txt

# 디렉토리 복사 (-r 옵션)
$ cp -r source_dir destination_dir

# 여러 파일을 디렉토리에 복사
$ cp file1.txt file2.txt target_dir/
-r: 디렉토리 전체 복사
-i: 덮어쓰기 전 확인
-v: 진행 상황 표시

mv

기초
파일이나 디렉토리를 이동하거나 이름을 변경합니다. 
# 파일 이동
$ mv file.txt /path/to/destination/

# 파일 이름 변경
$ mv old_name.txt new_name.txt

# 디렉토리 이동
$ mv source_dir /path/to/destination/

rm

기초
파일이나 디렉토리를 삭제합니다. 
# 파일 삭제
$ rm file.txt

# 여러 파일 삭제
$ rm file1.txt file2.txt

# 디렉토리 삭제
$ rm -r directory_name

# 강제 삭제 (주의!)
$ rm -rf directory_name
⚠️ 주의: rm -rf는 강제로 모든 것을 삭제하므로 매우 조심해서 사용하세요!

📝 텍스트 처리

cat

기초
파일의 내용을 출력합니다. 
# 파일 내용 출력
$ cat file.txt

# 여러 파일 내용 연결 출력
$ cat file1.txt file2.txt

# 줄 번호와 함께 출력
$ cat -n file.txt

grep

중급
텍스트에서 패턴을 검색합니다. 정규표현식을 사용하여 복잡한 패턴 매칭이 가능하며, 로그 분석, 코드 검색, 데이터 필터링에 필수적인 도구입니다. 
# 특정 문자열 검색
$ grep "error" log.txt

# 대소문자 구분 없이 검색
$ grep -i "ERROR" log.txt

# 줄 번호와 함께 표시
$ grep -n "pattern" file.txt

# 재귀적으로 디렉토리 검색
$ grep -r "function" *.js

# 정규표현식 사용
$ grep -E "^[0-9]+$" numbers.txt

# 매치되지 않는 줄 출력
$ grep -v "debug" log.txt

# 매치된 부분만 출력
$ grep -o "[0-9]\+\.[0-9]\+\.[0-9]\+\.[0-9]\+" access.log

head / tail

기초
파일의 앞부분(head) 또는 뒷부분(tail)을 출력합니다. 
# 파일의 첫 10줄 출력
$ head file.txt

# 파일의 마지막 10줄 출력
$ tail file.txt

# 실시간으로 추가되는 내용 모니터링
$ tail -f /var/log/syslog

# 특정 줄 수 지정
$ head -20 file.txt

⚙️ 프로세스 관리

ps

중급
실행 중인 프로세스 목록을 표시합니다. Process Status의 줄임말로, 시스템에서 실행 중인 프로그램들의 상태, 리소스 사용량, 프로세스 ID 등을 확인할 수 있습니다. 
# 현재 터미널의 프로세스
$ ps

# 모든 프로세스 자세히 보기
$ ps aux

# 특정 사용자의 프로세스
$ ps -u username

# 트리 구조로 프로세스 관계 표시
$ ps auxf

# 특정 프로세스 검색
$ ps aux | grep nginx

# 메모리 사용량 순으로 정렬
$ ps aux --sort=-%mem | head -10

# CPU 사용량 순으로 정렬
$ ps aux --sort=-%cpu | head -10

top / htop

중급
실시간으로 시스템 프로세스를 모니터링합니다. 
# 실시간 프로세스 모니터링
$ top

# 더 사용자 친화적인 인터페이스
$ htop
top 사용법:
  • q: 종료
  • k: 프로세스 종료
  • u: 특정 사용자 필터

kill

중급
프로세스를 종료합니다. 
# PID로 프로세스 종료
$ kill 1234

# 강제 종료
$ kill -9 1234

# 프로세스 이름으로 종료
$ killall firefox
주의: kill -9는 강제 종료이므로 데이터 손실 가능성이 있습니다.

🌐 네트워크 관리

ping

기초
네트워크 연결을 테스트합니다. 
# 구글 DNS 핑 테스트
$ ping google.com

# 4번만 핑 전송
$ ping -c 4 google.com

wget / curl

중급
인터넷에서 파일을 다운로드하거나 HTTP 요청을 보냅니다. 
# 파일 다운로드
$ wget https://example.com/file.zip

# HTTP GET 요청
$ curl https://api.example.com/data

# POST 요청
$ curl -X POST -d "data=value" https://api.example.com

🔥 중급 명령어 & 고급 기능

find

고급
파일 시스템에서 파일과 디렉토리를 검색합니다. 
# 이름으로 파일 검색
$ find /path -name "*.txt"

# 크기로 파일 검색
$ find /path -size +100M

# 수정 시간으로 검색
$ find /path -mtime -7

# 실행 권한이 있는 파일 검색
$ find /path -executable -type f

sed

고급
스트림 에디터로 텍스트를 변환하고 필터링합니다. 
# 문자열 치환
$ sed 's/old/new/g' file.txt

# 특정 줄 삭제
$ sed '2d' file.txt

# 파일 직접 수정
$ sed -i 's/old/new/g' file.txt

awk

고급
강력한 텍스트 처리 도구입니다. 
# 특정 열 출력
$ awk '{print $1}' file.txt

# 조건부 출력
$ awk '$3 > 100 {print $1, $3}' data.txt

# 합계 계산
$ awk '{sum += $2} END {print sum}' numbers.txt

파이프라인 (|)

중급
명령어들을 연결하여 복합적인 작업을 수행합니다. 
# 파일에서 특정 문자열 검색 후 정렬
$ grep "error" log.txt | sort

# 프로세스 검색 후 개수 세기
$ ps aux | grep firefox | wc -l

# 용량이 큰 파일 상위 10개
$ du -sh * | sort -hr | head -10

권한 관리

중급
파일과 디렉토리의 권한을 관리합니다. 
# 권한 변경 (chmod)
$ chmod 755 script.sh
$ chmod u+x file.txt

# 소유자 변경 (chown)
$ sudo chown user:group file.txt

# 권한 확인
$ ls -l file.txt
권한 숫자:
  • 4: 읽기(r)
  • 2: 쓰기(w)
  • 1: 실행(x)

🎯 고급 명령어

고급 사용자를 위한 강력하고 전문적인 Linux 명령어들입니다. 시스템 최적화와 고급 작업에 필수적입니다.

rsync

고급
효율적인 파일 동기화 및 백업 도구입니다. 네트워크를 통한 원격 동기화도 지원합니다.
로컬 동기화:
# 디렉토리 동기화 (삭제 옵션 포함)
$ rsync -av --delete source/ destination/

# 진행 상황 표시
$ rsync -av --progress source/ destination/
원격 동기화:
# 원격 서버로 업로드
$ rsync -avz local_folder/ user@server:/remote/path/

# 원격 서버에서 다운로드
$ rsync -avz user@server:/remote/path/ local_folder/

# SSH 포트 지정
$ rsync -avz -e "ssh -p 2222" source/ user@server:/dest/
-a: 아카이브 모드 (권한, 타임스탬프 보존)
-v: 상세 출력
-z: 압축 전송
--delete: 대상에서 소스에 없는 파일 삭제
--exclude: 특정 파일/디렉토리 제외
실무 활용:
  • 서버 백업 자동화
  • 웹사이트 배포
  • 대용량 파일 동기화
  • 증분 백업 구현

tar

고급
파일들을 하나의 아카이브로 묶거나 압축하는 도구입니다. 백업과 배포에 필수적입니다.
아카이브 생성:
# tar 파일 생성
$ tar -cvf archive.tar files/

# gzip 압축과 함께
$ tar -czvf archive.tar.gz files/

# bzip2 압축 (더 높은 압축률)
$ tar -cjvf archive.tar.bz2 files/

# 특정 파일 제외하고 압축
$ tar -czvf backup.tar.gz --exclude='*.log' --exclude='tmp/' project/
아카이브 해제:
# tar 파일 해제
$ tar -xvf archive.tar

# 특정 디렉토리에 해제
$ tar -xzvf archive.tar.gz -C /destination/path/

# 내용 확인만 (해제하지 않음)
$ tar -tzvf archive.tar.gz
-c: 아카이브 생성
-x: 아카이브 해제
-t: 내용 확인
-v: 상세 출력
-f: 파일 이름 지정
-z: gzip 압축/해제
-j: bzip2 압축/해제

xargs

고급
표준 입력으로부터 인수를 읽어 다른 명령어에 전달하는 도구입니다. 
# 파일 목록을 rm에 전달
$ find . -name "*.tmp" | xargs rm

# 병렬 처리 (4개 프로세스)
$ find . -name "*.jpg" | xargs -P 4 -I {} convert {} {}.webp

# 공백이 포함된 파일명 처리
$ find . -name "*.txt" -print0 | xargs -0 wc -l

# 확인 후 실행
$ echo "file1 file2 file3" | xargs -p rm
-I {}: 대체 문자열 지정
-P n: n개의 병렬 프로세스
-0: null 문자로 구분된 입력 처리
-p: 실행 전 확인

sort & uniq

고급
데이터 정렬과 중복 제거를 위한 강력한 도구들입니다.
Sort 활용:
# 숫자 정렬
$ sort -n numbers.txt

# 역순 정렬
$ sort -r file.txt

# 특정 열 기준 정렬 (2번째 열)
$ sort -k 2 data.txt

# 파일 크기 기준 정렬
$ du -sh * | sort -h
Uniq 활용:
# 중복 라인 제거
$ sort file.txt | uniq

# 중복 횟수 출력
$ sort file.txt | uniq -c

# 중복된 라인만 출력
$ sort file.txt | uniq -d
실무 예제:
# 로그에서 가장 많이 접근한 IP 찾기
$ awk '{print $1}' access.log | sort | uniq -c | sort -nr | head -10

# 가장 큰 파일 10개 찾기
$ find . -type f -exec du -h {} \; | sort -hr | head -10

jq

고급
JSON 데이터를 조작하고 필터링하는 강력한 도구입니다. API 작업에 필수적입니다.
기본 사용법:
# JSON 예쁘게 출력
$ curl -s https://api.github.com/users/octocat | jq .

# 특정 필드 추출
$ jq '.name' user.json

# 배열에서 특정 요소
$ jq '.[0].title' posts.json
고급 필터링:
# 조건부 필터링
$ jq '.[] | select(.age > 25)' users.json

# 복잡한 변환
$ jq 'map({name: .name, email: .contact.email})' data.json

# 그룹화 및 집계
$ jq 'group_by(.category) | map({category: .[0].category, count: length})' items.json
실무 활용:
  • API 응답 파싱
  • 설정 파일 조작
  • 로그 분석
  • 데이터 변환

tmux / screen

고급
터미널 멀티플렉서로 세션을 관리하고 백그라운드 작업을 수행합니다.
Tmux 기본 명령:
# 새 세션 시작
$ tmux new-session -s mysession

# 세션 목록 확인
$ tmux list-sessions

# 세션에 다시 연결
$ tmux attach-session -t mysession

# 세션에서 분리 (Ctrl+b, d)
# 새 윈도우 생성 (Ctrl+b, c)
# 윈도우 전환 (Ctrl+b, 0-9)
Screen 기본 명령:
# 새 세션 시작
$ screen -S mysession

# 세션 목록
$ screen -ls

# 세션 재연결
$ screen -r mysession

# 세션에서 분리 (Ctrl+a, d)
# 새 윈도우 (Ctrl+a, c)

🔧 시스템 관리

시스템 관리자를 위한 핵심 명령어들입니다. 서버 운영과 시스템 모니터링에 필수적입니다.

df / du

중급
디스크 사용량을 확인하는 명령어들입니다.
디스크 용량 확인 (df):
# 파일시스템 용량 확인
$ df -h
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        20G   15G  4.2G  78% /
/dev/sda2       100G   45G   50G  48% /home

# inode 사용량 확인
$ df -i

# 특정 파일시스템만
$ df -h /home
디렉토리 용량 확인 (du):
# 현재 디렉토리 용량
$ du -sh .

# 하위 디렉토리별 용량
$ du -sh */

# 가장 큰 디렉토리 10개
$ du -sh * | sort -hr | head -10

# 특정 깊이까지만
$ du -h --max-depth=2
🌍 실무 시나리오:

"디스크 용량이 부족하다는 알림이 왔을 때, 어떤 디렉토리가 가장 많은 공간을 차지하는지 빠르게 파악하세요."

systemctl

고급
systemd 서비스를 관리하는 명령어입니다. 최신 Ubuntu 시스템의 핵심입니다.
서비스 관리:
# 서비스 시작/중지/재시작
$ sudo systemctl start nginx
$ sudo systemctl stop nginx
$ sudo systemctl restart nginx
$ sudo systemctl reload nginx

# 서비스 상태 확인
$ systemctl status nginx

# 부팅 시 자동 시작 설정
$ sudo systemctl enable nginx
$ sudo systemctl disable nginx
시스템 정보:
# 모든 서비스 목록
$ systemctl list-units --type=service

# 실행 중인 서비스만
$ systemctl list-units --type=service --state=running

# 실패한 서비스
$ systemctl --failed

# 시스템 로그 확인
$ journalctl -u nginx -f
자주 관리하는 서비스:
  • nginx: 웹 서버
  • apache2: 웹 서버
  • mysql: 데이터베이스
  • postgresql: 데이터베이스
  • ssh: 원격 접속

crontab

고급
작업 스케줄링을 위한 도구입니다. 정기적인 백업, 로그 정리 등에 활용합니다.
크론탭 관리:
# 크론탭 편집
$ crontab -e

# 크론탭 목록 확인
$ crontab -l

# 크론탭 삭제
$ crontab -r
크론 문법 예제:
# 매일 오전 2시에 백업
0 2 * * * /home/user/backup.sh

# 매주 일요일 오전 3시에 로그 정리
0 3 * * 0 /usr/bin/find /var/log -name "*.log" -mtime +7 -delete

# 매 5분마다 시스템 상태 확인
*/5 * * * * /usr/bin/uptime >> /var/log/uptime.log

# 평일 오전 9시에 리포트 전송
0 9 * * 1-5 /home/user/send_report.sh
크론 시간 형식:

분(0-59) 시(0-23) 일(1-31) 월(1-12) 요일(0-7)

  • *: 모든 값
  • */n: n마다
  • a-b: a부터 b까지
  • a,b,c: a, b, c

iotop / iftop

고급
I/O 및 네트워크 사용량을 실시간 모니터링하는 도구들입니다. 
# 디스크 I/O 모니터링
$ sudo iotop

# 네트워크 대역폭 모니터링
$ sudo iftop

# 메모리 사용량 상세 확인
$ free -h
$ cat /proc/meminfo

# 시스템 부하 확인
$ uptime
$ w
$ who

firewall (ufw)

고급
Ubuntu의 기본 방화벽 관리 도구입니다. 
# 방화벽 활성화/비활성화
$ sudo ufw enable
$ sudo ufw disable

# 포트 허용/차단
$ sudo ufw allow 22/tcp    # SSH 허용
$ sudo ufw allow 80/tcp    # HTTP 허용
$ sudo ufw allow 443/tcp   # HTTPS 허용
$ sudo ufw deny 23/tcp     # Telnet 차단

# IP 주소 기반 규칙
$ sudo ufw allow from 192.168.1.100
$ sudo ufw deny from 192.168.1.200

# 상태 확인
$ sudo ufw status verbose

📜 셸 스크립팅 기초

반복 작업을 자동화하고 복잡한 작업을 단순화하는 셸 스크립팅을 학습합니다.

기본 스크립트 구조

고급
셸 스크립트의 기본 구조와 작성 방법을 익힙니다.
기본 구조:
#!/bin/bash
# 스크립트 설명

# 변수 선언
NAME="Ubuntu"
VERSION="22.04"

# 함수 정의
show_info() {
    echo "운영체제: $NAME $VERSION"
    echo "현재 시간: $(date)"
    echo "현재 사용자: $USER"
}

# 메인 실행부
echo "시스템 정보 스크립트 시작"
show_info
echo "스크립트 완료"
실행 권한 부여:
# 실행 권한 추가
$ chmod +x myscript.sh

# 스크립트 실행
$ ./myscript.sh

조건문과 반복문

고급
스크립트에서 조건부 실행과 반복 작업을 구현합니다.
조건문 (if-else):
#!/bin/bash

# 파일 존재 확인
if [ -f "/etc/passwd" ]; then
    echo "파일이 존재합니다."
else
    echo "파일이 존재하지 않습니다."
fi

# 숫자 비교
NUM=10
if [ $NUM -gt 5 ]; then
    echo "$NUM은 5보다 큽니다."
elif [ $NUM -eq 5 ]; then
    echo "$NUM은 5와 같습니다."
else
    echo "$NUM은 5보다 작습니다."
fi
반복문 (for, while):
# for 반복문
for i in {1..5}; do
    echo "숫자: $i"
done

# 파일 목록 처리
for file in *.txt; do
    if [ -f "$file" ]; then
        echo "처리 중: $file"
        # 파일 처리 작업
    fi
done

# while 반복문
COUNTER=1
while [ $COUNTER -le 5 ]; do
    echo "카운터: $COUNTER"
    COUNTER=$((COUNTER + 1))
done

실무 스크립트 예제

고급
실제 업무에서 활용할 수 있는 유용한 스크립트 예제들입니다.
1. 백업 스크립트:
#!/bin/bash

# 백업 설정
SOURCE_DIR="/home/user/important_data"
BACKUP_DIR="/backup"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_NAME="backup_$DATE.tar.gz"

# 백업 디렉토리 확인
if [ ! -d "$BACKUP_DIR" ]; then
    mkdir -p "$BACKUP_DIR"
fi

# 백업 실행
echo "백업 시작: $(date)"
tar -czf "$BACKUP_DIR/$BACKUP_NAME" "$SOURCE_DIR"

if [ $? -eq 0 ]; then
    echo "백업 완료: $BACKUP_DIR/$BACKUP_NAME"
else
    echo "백업 실패!"
    exit 1
fi

# 30일 이상 된 백업 파일 삭제
find "$BACKUP_DIR" -name "backup_*.tar.gz" -mtime +30 -delete
echo "백업 작업 완료: $(date)"
2. 시스템 모니터링 스크립트:
#!/bin/bash

# 임계값 설정
CPU_THRESHOLD=80
MEM_THRESHOLD=90
DISK_THRESHOLD=85

# CPU 사용률 확인
CPU_USAGE=$(top -bn1 | grep "Cpu(s)" | awk '{print $2}' | cut -d'%' -f1)
CPU_USAGE=${CPU_USAGE%.*}  # 소수점 제거

# 메모리 사용률 확인
MEM_USAGE=$(free | grep Mem | awk '{printf "%.0f", $3/$2 * 100.0}')

# 디스크 사용률 확인
DISK_USAGE=$(df -h / | awk 'NR==2{print $5}' | cut -d'%' -f1)

# 로그 파일
LOG_FILE="/var/log/system_monitor.log"

# 경고 메시지 생성
echo "[$(date)] CPU: ${CPU_USAGE}%, MEM: ${MEM_USAGE}%, DISK: ${DISK_USAGE}%" >> "$LOG_FILE"

# 임계값 초과 시 알림
if [ "$CPU_USAGE" -gt "$CPU_THRESHOLD" ]; then
    echo "WARNING: CPU usage is ${CPU_USAGE}%" | tee -a "$LOG_FILE"
fi

if [ "$MEM_USAGE" -gt "$MEM_THRESHOLD" ]; then
    echo "WARNING: Memory usage is ${MEM_USAGE}%" | tee -a "$LOG_FILE"
fi

if [ "$DISK_USAGE" -gt "$DISK_THRESHOLD" ]; then
    echo "WARNING: Disk usage is ${DISK_USAGE}%" | tee -a "$LOG_FILE"
fi

📚 종합 참고 자료

🔍 명령어 치트시트

네비게이션

  • pwd - 현재 디렉토리 경로
  • ls -la - 상세 파일 목록
  • cd ~ - 홈 디렉토리로
  • cd - - 이전 디렉토리로

파일 조작

  • cp -r source dest - 디렉토리 복사
  • mv old new - 이동/이름변경
  • rm -rf folder - 강제 삭제
  • find . -name "*.txt" - 파일 검색

텍스트 처리

  • grep -r "pattern" . - 재귀 검색
  • sed 's/old/new/g' - 문자열 치환
  • awk '{print $1}' - 첫번째 컬럼
  • sort | uniq -c - 정렬 후 중복 제거

시스템

  • ps aux | grep process - 프로세스 검색
  • df -h - 디스크 사용량
  • du -sh * - 디렉토리별 용량
  • systemctl status service - 서비스 상태

🚀 실무 활용 시나리오

📊 로그 분석

# 에러 로그 분석 및 통계
grep "ERROR" /var/log/app.log | \
awk '{print $4}' | sort | uniq -c | sort -nr

# 특정 시간대 로그 필터링
sed -n '/2024-08-23 14:00/,/2024-08-23 15:00/p' access.log

# 접근 빈도 높은 IP 상위 10개
awk '{print $1}' access.log | sort | uniq -c | \
sort -nr | head -10

🔧 시스템 모니터링

# 시스템 리소스 종합 체크
echo "=== CPU Usage ==="
top -bn1 | grep "Cpu(s)"
echo "=== Memory Usage ==="
free -h
echo "=== Disk Usage ==="
df -h
echo "=== Top Processes ==="
ps aux --sort=-%cpu | head -5

📁 대용량 파일 관리

# 큰 파일 찾기 (100MB 이상)
find / -type f -size +100M -exec ls -lh {} \; 2>/dev/null

# 디렉토리 용량 분석
du -h --max-depth=1 | sort -hr

# 오래된 로그 파일 정리 (30일 이상)
find /var/log -name "*.log" -mtime +30 -delete

⚡ 성능 최적화 팁

🔍 명령어 최적화

  • 파이프 순서: 가장 필터링이 많이 되는 명령어를 앞에
  • grep vs find: 텍스트 검색은 grep, 파일명은 find
  • xargs 활용: 대량 파일 처리 시 병렬 처리로 속도 향상
  • 메모리 고려: sort 대신 필요한 경우에만 사용

📈 성능 모니터링

  • time command - 명령어 실행 시간 측정
  • iostat 1 - I/O 성능 실시간 모니터링
  • htop - 프로세스별 리소스 사용량
  • iotop - 디스크 I/O 사용량

🔒 보안 및 권한 관리

파일 권한 (chmod)

755 rwxr-xr-x (소유자: 모든권한, 그룹/기타: 읽기+실행)
644 rw-r--r-- (소유자: 읽기+쓰기, 그룹/기타: 읽기만)
600 rw------- (소유자만 읽기+쓰기)

보안 체크리스트

  • 민감한 파일은 600 권한 (소유자만 접근)
  • 실행 파일은 755, 설정 파일은 644
  • sudo 사용 시 최소 권한 원칙 적용
  • 정기적인 로그 모니터링
  • 불필요한 서비스 비활성화

🛠️ 문제해결 가이드

❌ "Permission denied" 오류

원인: 파일/디렉토리에 대한 권한이 없음

해결:

# 권한 확인
ls -l filename

# 권한 변경
chmod 755 filename

# 소유자 변경 (필요시)
sudo chown username:groupname filename

💾 "No space left on device" 오류

해결 단계:

# 1. 디스크 사용량 확인
df -h

# 2. 큰 파일/디렉토리 찾기
du -h / | sort -hr | head -20

# 3. 불필요한 파일 정리
sudo apt clean
sudo journalctl --vacuum-time=7d

# 4. 로그 파일 정리
sudo find /var/log -type f -name "*.log" -mtime +7 -delete

🐌 시스템이 느려질 때

# CPU 사용률 높은 프로세스 찾기
ps aux --sort=-%cpu | head -10

# 메모리 사용률 높은 프로세스
ps aux --sort=-%mem | head -10

# I/O 대기 시간 확인
iostat -x 1

# 네트워크 연결 상태
netstat -tuln
