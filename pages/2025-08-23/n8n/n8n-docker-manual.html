⚡n8n Docker 설치

🚀 빠른 시작 (단일 컨테이너)

# n8n 컨테이너 실행 docker run -it --rm \ --name n8n \ -p 5678:5678 \ -v ~/.n8n:/home/node/.n8n \ n8nio/n8n

🔧 Docker Compose 설정

# docker-compose.yml version: '3.8' services: n8n: image: n8nio/n8n restart: always ports: - "5678:5678" environment: - N8N_BASIC_AUTH_ACTIVE=true - N8N_BASIC_AUTH_USER=admin - N8N_BASIC_AUTH_PASSWORD=password - N8N_HOST=localhost - N8N_PORT=5678 - N8N_PROTOCOL=http volumes: - n8n_data:/home/node/.n8n - ./shared:/home/node/shared volumes: n8n_data:
# Docker Compose로 실행 docker-compose up -d # 로그 확인 docker-compose logs -f n8n # 중지 docker-compose down
접속 URL: http://localhost:5678
기본 인증: admin / password

📚n8n 기본 사용법

🎯 핵심 개념

  • 워크플로우: 자동화 프로세스
  • 노드: 개별 작업 단위
  • 트리거: 워크플로우 시작점
  • 연결: 노드 간 데이터 흐름

⚙️ 주요 노드 타입

  • Trigger: Webhook, Cron, Manual
  • Regular: HTTP Request, Function
  • Credentials: API 키 관리

🔧 환경 변수 설정

# .env 파일 N8N_BASIC_AUTH_ACTIVE=true N8N_BASIC_AUTH_USER=your_username N8N_BASIC_AUTH_PASSWORD=your_password N8N_HOST=0.0.0.0 N8N_PORT=5678 N8N_PROTOCOL=http WEBHOOK_URL=http://localhost:5678 N8N_ENCRYPTION_KEY=your_encryption_key

📁 데이터 지속성

# 볼륨 마운트 -v ~/.n8n:/home/node/.n8n -v ./workflows:/home/node/workflows -v ./credentials:/home/node/credentials

🎨실용적인 워크플로우 예제

📧 이메일 알림 자동화

🕐 Cron Trigger → 🌐 HTTP Request → 🔍 IF 조건 → 📧 Gmail

매일 오전 9시에 서비스 상태를 확인하고 문제가 있으면 이메일 발송

💬 슬랙 봇 통합

🎣 Webhook → ⚙️ Function → 🗄️ Database → 💬 Slack

외부 이벤트를 받아 데이터를 처리하고 슬랙 채널에 알림

📊 데이터 동기화

🕐 Schedule → 📊 Google Sheets → 🔄 Transform → 🗄️ PostgreSQL

구글 시트 데이터를 정기적으로 데이터베이스와 동기화

🔧고급 설정

🔒 보안 설정

# HTTPS 설정 environment: - N8N_PROTOCOL=https - N8N_SSL_KEY=/path/to/key.pem - N8N_SSL_CERT=/path/to/cert.pem

📈 성능 최적화

  • 메모리 제한: --memory=2g
  • CPU 제한: --cpus=2
  • 로그 레벨: N8N_LOG_LEVEL=info

🔄 백업 및 복원

# 데이터 백업 docker cp n8n:/home/node/.n8n ./backup/ # 데이터 복원 docker cp ./backup/.n8n n8n:/home/node/

🔍문제해결

❌ 일반적인 문제

  • 포트 충돌: 5678 포트 변경
  • 권한 오류: 볼륨 권한 확인
  • 메모리 부족: Docker 메모리 증설

🔍 디버깅 명령어

# 컨테이너 상태 확인 docker ps -a # 로그 확인 docker logs n8n # 컨테이너 내부 접속 docker exec -it n8n /bin/sh # 리소스 사용량 docker stats n8n
